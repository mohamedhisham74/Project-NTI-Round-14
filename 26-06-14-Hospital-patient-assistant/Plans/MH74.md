## 2. Vector Layer (The Technical Core of RAG)

### 2.1 Pipeline Overview

```
Raw Docs ──► Loader ──► Chunker ──► Embedder ──► Vector DB
(PDF/DOCX)              (512/50)   (1536-dim)   (+ metadata)
                                                     │
Query ──► Embedder ──► ANN Search (top-k) ──► Reranker ──► top-3
                       (cosine, HNSW)        (cross-enc)
```

### 2.2 Embedding Configuration

```python
# Embedding model selection
EMBEDDING_CONFIG = {
    "model": "text-embedding-3-small",   # or multilingual-e5 for Arabic
    "dimensions": 1536,
    "max_tokens": 8191,
    "normalize": True,                    # for cosine similarity
    "batch_size": 100,
}
```

| Decision | Choice | Why |
|---|---|---|
| Model | `text-embedding-3-small` / `multilingual-e5-large` | Arabic + English support |
| Dimensions | 1536 | Balance: accuracy vs storage/speed |
| Distance | Cosine | Standard for normalized embeddings |
| Index | HNSW | Fast Approximate Nearest Neighbor |

### 2.3 Chunking Strategy

```python
from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=512,           # tokens per chunk
    chunk_overlap=50,         # context continuity
    separators=["\n\n", "\n", ".", "،", " "],  # Arabic-aware
    length_function=count_tokens,
)
```

> **Why 512/50?** Small enough for precise retrieval, large enough to keep a doctor's full schedule or one policy intact. 50-token overlap prevents cutting context mid-sentence.

### 2.5 Hybrid Retrieval + Reranking

```python
async def retrieve(query: str, filters: dict, k: int = 10):
    # 1. Dense (semantic) search
    q_vec = await embed(query)
    dense = vector_db.search(q_vec, top_k=k, filter=filters,
                             metric="cosine")

    # 2. Sparse (keyword) search — BM25
    sparse = bm25_index.search(query, top_k=k, filter=filters)

    # 3. Fuse results (Reciprocal Rank Fusion)
    fused = reciprocal_rank_fusion(dense, sparse)

    # 4. Rerank with cross-encoder for true relevance
    reranked = cross_encoder.rerank(query, fused, top_k=3)
    return reranked
```

```
WHY EACH STEP:
  Dense    → catches meaning ("heart doctor" ≈ "cardiologist")
  Sparse   → catches exact terms (doctor names, dates, codes)
  Fusion   → combines strengths of both
  Rerank   → final precision pass (Precision@3 ≥ 90%)
```

### 2.6 Vector DB Comparison

| DB | Best For | Notes |
|---|---|---|
| **ChromaDB** | PoC / prototyping | Easy, embedded, local |
| **Qdrant** | Production | Fast, filtering, scalable, on-prem |
| **FAISS** | Max speed, in-memory | No metadata filtering natively |
| **pgvector** | Already on PostgreSQL | Unifies vector + relational |

> **Recommendation:** Qdrant for production (rich metadata filtering + on-prem for medical data compliance).

## 3. Framework Layer (LangGraph State Machine)

### 3.1 Why LangGraph (not plain chains)

```
Plain Chain (LangChain)        StateGraph (LangGraph)
───────────────────            ──────────────────────
Linear A→B→C                   Cyclic, conditional
No shared state                Shared typed state
Hard to branch                 Native conditional edges
No human-in-the-loop           Built-in interrupts
                               Checkpointing / persistence
```

A hospital flow is **not linear** — it branches (emergency vs normal), loops (clarification), and pauses (human handoff). LangGraph models this as a state machine.

### 3.2 State Definition

```python
from typing import TypedDict, Annotated, Literal
from langgraph.graph import StateGraph, END
from langgraph.graph.message import add_messages

class HospitalState(TypedDict):
    messages: Annotated[list, add_messages]   # conversation
    patient_id: str | None
    intent: str | None                         # from Triage
    severity: int                              # 1-10
    is_emergency: bool
    target_agent: str | None                   # routing decision
    retrieved_context: list                    # RAG results
    confidence: float
    needs_human: bool                          # handoff flag
    response: str | None
```

### 3.3 Graph Construction

```python
graph = StateGraph(HospitalState)

# --- Nodes (each is an agent or function) ---
graph.add_node("guardrails_in", guardrails_input)
graph.add_node("triage",        triage_agent)
graph.add_node("rag",           rag_agent)
graph.add_node("scheduling",    scheduling_agent)
graph.add_node("recommendation",recommendation_agent)
graph.add_node("analytics",     analytics_agent)
graph.add_node("human_handoff", human_handoff_agent)
graph.add_node("emergency",     emergency_handler)
graph.add_node("guardrails_out",guardrails_output)

# --- Entry ---
graph.set_entry_point("guardrails_in")
graph.add_edge("guardrails_in", "triage")

# --- Conditional routing after triage ---
def route_after_triage(state: HospitalState) -> str:
    if state["is_emergency"]:
        return "emergency"
    if state["confidence"] < 0.80:
        return "human_handoff"
    return state["target_agent"]   # "rag" | "scheduling" | ...

graph.add_conditional_edges(
    "triage",
    route_after_triage,
    {
        "emergency":      "emergency",
        "human_handoff":  "human_handoff",
        "rag":            "rag",
        "scheduling":     "scheduling",
        "recommendation": "recommendation",
        "analytics":      "analytics",
    }
)

# --- All agents converge to output guardrails ---
for node in ["rag","scheduling","recommendation","analytics","emergency"]:
    graph.add_edge(node, "guardrails_out")

graph.add_edge("guardrails_out", END)
graph.add_edge("human_handoff", END)   # paused for human

# --- Compile with persistence (checkpointing) ---
app = graph.compile(checkpointer=PostgresSaver(...))
```

### 3.5 An Agent Node (Internals)

```python
async def rag_agent(state: HospitalState) -> dict:
    query = state["messages"][-1].content

    # Agent REASONS about which tools to call
    plan = await llm.bind_tools([search_kb, rerank, verify_answer]) \
                    .ainvoke(state["messages"])

    # Execute tool calls
    context = await execute_tools(plan.tool_calls)

    # Generate grounded answer
    answer = await generate_answer(query, context)

    # Self-check faithfulness (guardrail inside agent)
    faith = await verify_answer(answer, context)
    if faith < 0.95:
        return {"needs_human": True, "confidence": faith}

    return {
        "response": answer,
        "retrieved_context": context,
        "confidence": faith,
    }
```

## 4. Agent–Tool Technical Spec

### 4.1 A Tool Definition (typed, async)

```python
from langchain_core.tools import tool
from pydantic import BaseModel, Field

class SlotQuery(BaseModel):
    department: str = Field(description="Medical department")
    preferred_date: str = Field(description="ISO date YYYY-MM-DD")
    visit_type: Literal["first_visit", "follow_up"]

@tool(args_schema=SlotQuery)
async def find_slots(department: str, preferred_date: str,
                     visit_type: str) -> list[dict]:
    """Find available appointment slots. Deterministic — no LLM."""
    return await hms_api.get_available_slots(
        dept=department, date=preferred_date, type=visit_type
    )
```

### 4.2 Agents vs Tools (technical line)

```
AGENT  = LLM + reasoning + tool-binding  →  decides WHAT to do
TOOL   = pure async function             →  executes ONE thing

Cost:    Agent = $$ (LLM tokens)   |   Tool = ~$0 (DB/API call)
Latency: Agent = 500-2000ms        |   Tool = 10-100ms
```

| Observability | LangSmith + Grafana + Prometheus | Tracing + metrics |
| Eval | RAGAS + custom suites | Quality gates |

## 🎯 Technical Summary

```
VECTOR     → Qdrant + HNSW + hybrid (dense+sparse) + cross-encoder rerank
FRAMEWORK  → LangGraph StateGraph: typed state, conditional edges,
             checkpointing, human-in-the-loop interrupts
ARCHITECTURE → 6 layers: Client → Gateway/Guardrails → Orchestration
             → Agents → Tools → (Vector | Memory | Hospital Systems)

Key principle: Agents reason (LLM, expensive),
               Tools execute (functions, cheap, fast).
```
